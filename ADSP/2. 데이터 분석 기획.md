### 2.1.1 분석기획 방향성 도출

#### 분석 기획

분석기획이란 실제 분석을 하기전에 수행할 과제를 정의하고, 의도했던 결과를 도출할 수 있도록 적절하기 관리하는 방안을 사전에 계획하는 일련의 작업



분석 기획을 하기 위해서는 3가지 역량이 필요

* 수학/통계학적 지식
* 정보기술
* 해당 비즈니스에 대한 이해와 전문성



분석 대상과 분석 방법에 따라 4가지의 분석으로 나뉨

![image-20201117200130263](C:\Users\multicampus\PycharmProjects\algo\ADSP\image-20201117200130263.png)

* 최적화 (Optimization) : 해결해야 할 문제를 알고 있고 분석 방법도 알고 있는 경우
* 솔루션(Solution): 해결할 문제는 알지만, 분석 방법을 모르는 경우
* 통찰(Insight) : 대상이 명확하게 무엇인지 모르는 경우 , 기존 분석방법 활용
* 발견(Discovery) : 분석의 대상 자체를 새롭게 도출함



목표 시점별 분석 기획 방안

### 2.2.2 분석 방법론

##### 개요

기업내에서 데이터 분석을 효과적으로 하기 위해 상세한 절차, 방법, 도구와 기법, 템플릿과 산출물이 있어야만 한다.



##### 데이터 기반 의사결정의 필요성

합리적인 의사결정을 하기 위해선 감이나 경험보다는 데이터 기반의 의사결정이 필요하다. 감이나 경험에 의지할경우 고정관점, 편향된 생각, 프레이밍 효과 등으로 잘못된 결정을 할 수있다.

`프레이밍 효과` : 동일한 상황임에도 개인의 판단에 따라 결정이 달라질 수 있는 현상



##### 특성에 따른 모델

* 폭포수 모델
  * 단계를 순차적으로 진행하는 방법
* 프로토타입 모델
  * 일부분을 우선 개발하여 사용자에게 제공한뒤, 사용자의 피드백을 받아 개선작업하는 모델
* 나선형 모델
  * 반복을 통해 점증적으로 개발하는 모델



 #### KDD 분석방법론

KDD(Knowledge Discovery in Databases)은 데이터로 부터 패턴이나 지식을 찾기 위해 체계적으로 정리한 데이터 마이닝 프로세스



KDD의 분석 절차

1. 데이터셋 선택
   * 데이터마이닝에 필요한 target data를 구성하는 단계
2. 데이터 전처리
   * 추출된 데이터셋에서 의미있는 데이터로 가공하는 단계
3. 데이너 변환
   * 전처리과정을 겪을 데이터를 학습용과 검증용으로 나누는 단계
4. 데이터 마이닝
   * 분석 목적에 맞는 데이터 마이닝 기법을 선택하고 적절한 알고리즘을 적용하는 단계
5. 데이터 마이닝 결과 평가
   * 데이터 마이닝 결과에 대한 해석과 평가



#### CRISP-DM 분석 방법론

##### CRISP - DM의 4가지 레벨

CRISP - DM은 계층적 프로세스 모델로써 4가지 레벨 구조로 구성된다.

![image-20201117211144066](C:\Users\multicampus\PycharmProjects\algo\ADSP\image-20201117211144066.png)



CRISP-DM의 4가지 레벨

* Phases(최상위 단계)
* Generic Tasks(일반화 데스크) : 데이터마이닝의 단일 프로세스를 완전하게 수행하는단위
* Specialized Tasks(세분화 테스크) : 구체적인 수행 레벨
* Process Instances(프로세스 실행)



##### CRISP-DM 의 6단계 프로세스

CRISP-DM 프로세스는 6단계로 구성되어있음

![image-20201117211712215](C:\Users\multicampus\PycharmProjects\algo\ADSP\image-20201117211712215.png)

| 단계                             | 내용                                                    | 수행업무                   |
| -------------------------------- | ------------------------------------------------------- | -------------------------- |
| 업무이해(Business Understanding) | 프로젝트의 목적과 요구사항을 이해하는 단계              | 업무 목적 파악, 목표 설정  |
| 데이터 이해(Data Understanding)  | 데이터를 수집, 속성을 이해하는 단계                     | 데이터 수집 및 분석        |
| 데이터 준비(Data Preparation)    | 적합한 데이터를 편성하는 단계                           | 데이터 정제, 데이터셋 선택 |
| 모델링(Modeling)                 | 모델링 기법과 알고리즘 최적화, 모델의 과적합문제를 확인 | 모델기법선택, 모델 평가    |
| 평가(Evaluation)                 | 프로젝트 목적에 부합하는지 평가                         | 분석 결과 평가             |
| 전개(Deployment)                 | 모델을 실 업무에 적용                                   | 프로젝트 리뷰, 보고서 작성 |
