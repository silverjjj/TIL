### 1.1 데이터와 정보

#### 데이터의 유형

| 구분          | 형태       | 예                    | 특징                              |
| ------------- | ---------- | --------------------- | --------------------------------- |
| 정성적 데이터 | 언어, 문자 | 회사 매출이 증가함 등 | 저장, 검색, 분석에 많은 비용      |
| 정량적 데이터 | 수치, 도형 | 나이, 몸무게, 주가 등 | 정형화된 데이터로 비용소모가 적음 |

정성적 데이터 : 주관적 내용, 통계분석이 어려움

정량적 데이터 : 객관적 내용, 통계분석이 쉬움



#### 암묵지 vs 형식지

##### 암묵지(tackit knowledge) 

개인에게 체화되어 있지만 겉으로 드러나지 않는 지식으로서 사회적으로 중요하지만 다른 사람에게 공유되기 어려움.

ex) 자전거타기, 김치 담그기

개인에게 축적된 내면화된 지식으로 조직의 지식으로 공통화됨 (내면화 => 공통화)



##### 형식지(explicit knowledge)

문서나 메뉴얼처럼 형식화된 지식으로서 전달과 공유가 용이함

ex) 교과서, 비디오, 영화 등

언어, 기호 등으로 표출화된 지식으로 개인의 지식으로 연결화(표출화 => 연결화)



#### DIKW 피라미드

Data => Infomation => Knowledge => Wisdom 으로 데이터로부터 정보, 지식을 거쳐 지혜를 얻어내는 과정을 계층화한것

* Data : 개별 데이터로는 의미가 없는 객관적인사실
* Information : 데이터 가공에 따라 데이터간 관계속에서 의미를 도출한것
* Knowledge : 도출된 정보 + 개인적인 경험을 결합시켜 지식으로 내재화한것
* Wisdom : 지식의 축적과 아이디어를 바탕으로 도출되는것

데이터에서 시작해서 데이터 가공에 따라 정보가 되고, 정보가 경험과 결합되어 지식이 되고 지식과 아이디어가 결합되어 지혜가됨



### 1.2 데이터베이스 정의와 특징

#### 데이터베이스 정의

데이터베이스는 시간에 따라 정의가 변화됨



##### 1차(정형데이터 관리)

체계적이로 정리되어 개별적으로 접근할 수 있는 독립된 저작물, 데이터

 

##### 2차(빅데이터의 출현으로 비정형데이터 포함)

복수 이용자의 요구에 대응해서 데이터를 받아들이고 용도와 방법에 따라 이용할 수 있는 정보의 집합체



### 데이터베이스의 특징

* 통합된 데이터(Integrated data)
  * 동일한 내용은 존재하지 않음
* 저장된 데이터(stored data)
  * 컴퓨터가 접근할 수 있는 저장매체에 저장되는 것을 의미
* 공용 데이터(shared data)
  * 여러 사용자가 서로 다른목적으로 데이터를 공동으로 이용한다는 의미
* 변화되는 데이터(changable data)
  * 데이터베이스에 저장되있는 내용은 곧 현 시점의 상태임
  * 이 상태는 데이터의 삽입, 갱신, 삭제에 다라 항상 변화함



### 1.3 데이터베이스의 활용

#### 약어의 의미

* OLTP(On-Line Transaction Processing)
  * 여러 단말에서 보내온 메시지에 따라 호스트 컴퓨터가 데이터베이스를 접근하고, 바로 처리 결과를 돌려보내는 형태
  * 즉, 데이터를 수시로 갱신하는 프로세싱
* OLAP(On-Line Analytical Processing)
  * 정보위주의 분석처리를 의미한다
  * 비즈니스 관점에서 데이터에 접근하여 의사 결정에 활용할수 있는 정보를 얻을 수 있는 기술
  * 쉽게말해서 데이터의 조회라고 할 수 있다.
* CRM(Customer Relationship Management)
  * `고객관계관리`라고 하며, 기업이 고객과 관련된 자료를 분석해, 고객 특성에 맞게 마케팅 활동을 하는 과정
* SCM(Supply Chain Management)
  * `공급망 관리`라고 하며, 기업에서 사용하는 모든 소재들의 공급망 단계를 최적화하는것
* ERF(Enterprise Resource Plannning)
  * 경영자원을 하나의 통합 시스템으로 재구축
* BI(Business Intelligence)
  * 데이터를 활용하여 기업의 의사결정에 활용하는 일련의 프로세스
* RTE(Real- Time Enterprise)
  * 회사의 전 부문의 정보를 하나로 통합



## 2장 데이터의 가치와 미래

### 2.1 빅데이터의 이해

#### 3V 란 ??

3V는 빅데이터를 바라보는 관점이다.

Volume(양), Variety(다양성), Velocity(속도)라고 하며, 각각

Volume : 데이터 규모의 측면

Variety : 데이터의 유형과 소스 측면

Velocity : 데이터의 수집과 처리 측면

으로 의미로 하여 빅데이터를 바라보는 관점의 의미를 내포한다.



그리고 빅데이터를 바라보는 관점이 점점 변화하고 있는데, 데이터를 중점적으로 보는 3V를 벗어나서 데이터를 처리, 저장, 분석하는 기술관점에서 바라보고, 현재는 Data를 직접 다루는 인재관점에서 바라본다.

다시 말해서, 빅데이터의 범주가 데이터변화 ==> 기술 변화 ==> 인재 변화로 점점 확대되고 있다.





#### 빅데이터의 비유

| 비유     | 기대                                                         |
| -------- | ------------------------------------------------------------ |
| 석탄(철) | 사회 전반에 혁명적 변화를 가져올것이라 기대                  |
| 원유     | 사회 전반의 생산성을 향상시키고 새로운 산업을 만들어냄       |
| 렌즈     | 현미경 렌즈를 통해 생물학이 발전한것처럼 데이터를 통해 산업이 발전 |
| 플랫품   | 공동 활용의 목적으로 구축된 유무형의 구조물로써 다양한 분야에 플랫폼 역할 |



#### 빅데이터에서 중요시 하는 부분의 변화

사전 처리 ==> 사후 처리

표본 조사 ==> 전수 조사

질 ==> 양

인과관계 ==> 상관관계

  * 사람들간의 관계를 파악할때 사용



### 2.4 위기 요인과 통제 방안

## 3장 가치 창조를 위한 데이터 사이언스와 전략 인사이트

### 3.1 빅데이터 분석과 전략 인사이트

#### 분석 어플리케이션

| 산업       | 일차원적 분석 어플리케이션                              |
| ---------- | ------------------------------------------------------- |
| 금융서비스 | 신용점수 산정, 가격책정, 사기탐지, 프로그램 트레이딩 등 |
| 병원       | 가격책정, 고객로열티, 수익관리                          |
| 에너지     | 트레이딩, 공급/수요예측                                 |
| 정부       | 사기탐지, 사례관리, 범죄방지, 수익최적화                |



#### 일차적인 분석과 전략도출 가치기반

##### 일차적인 분석의 문제점

* 일차원적 분석으로는 환경변화와 같은 큰 변화에 대응하기 힘들다. 즉, 새로운 기회를 포착하기 어렵다.

##### 전략도출 가치기반

* 전략적인 분석을 통해 사업에서 `중요한 기회`를 얻을수 있다.
* 처음에는 일차원적인 분석을 통해 경험을 쌓지만, 더 큰 성공을 위해 `활용범위를 더 넓고 전략적`으로 변화시켜야한다.
* 차별화를 만들때는 `전략적 인사이트를 주는 가치 분석 단계`로 나아가야한다.



### 3.2 전략 인사이트 도출을 위한 필요 역량

#### 데이터 사이언스의 의미와 역할

* 의미
  * 데이터 사이언스란 데이터로부터 의미있는 정보를 추출해내는 학문이다.
* 역할 
  * 비즈니스 성과에 좌우하는 이슈에 답하고 사업 성과를 견인해 나갈 수 있어야한다.
  * 데이터 사이언티스트는 스토리텔링, 소통, 창의력, 열정, 직관력 등을 보유해야한다.



#### 데이터 사이언스의 구성요소

* 분석적 영역
  * 수학, 머신러닝 등의 분석모델
* 데이터 처리 및 it 영역
  * 프로그래밍, 데이터 엔지니어링
* 비즈니스 컨설팅 영역
  * 커뮤니케이션, 프레젠테이션, 스토리탤링 등



#### 데이터 사이언티스트의 요구역량

* Hard Skill
  * 빅데이터에 대한 이론적 지식
  * 분석 기술에 대한 숙련
* Soft Skill
  * 통찰력 있는 분석
  * 설득력 있는 전달
  * 다분야간 협력
